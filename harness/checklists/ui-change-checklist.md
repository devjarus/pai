# UI Change Checklist

Use this when the change touches navigation, product copy, Brief presentation, Program surfaces, or trust UX.

- [ ] The change does not reintroduce information-architecture drift.
- [ ] Deprecated product nouns like `job`, `swarm`, or `schedule` do not become primary UI labels without a decision log.
- [ ] Mobile readability was considered for the touched screen or artifact.
- [ ] Recommendation remains visually first in any Brief-like output.
- [ ] Evidence and memory assumptions are visible somewhere appropriate in the experience.
- [ ] Advanced complexity stays hidden by default unless the screen is explicitly advanced or admin-oriented.
- [ ] Threads are not presented as the primary product object.
