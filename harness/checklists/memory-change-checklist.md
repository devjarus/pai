# Memory Change Checklist

Use this when the change touches beliefs, episodes, memory retrieval, provenance, correction, contradiction handling, or trust UX.

- [ ] Origin and provenance are preserved for affected memory paths.
- [ ] Observed facts and inferred claims remain distinguishable.
- [ ] Corrected or invalidated beliefs are suppressed from later recommendation paths where appropriate.
- [ ] Contradiction handling is not silently bypassed or weakened.
- [ ] Memory trust implications were reviewed in the UI or operator-facing output, not only in storage code.
- [ ] Confidence, freshness, or stability changes are intentional and documented.
- [ ] Evidence pack states what memory path was checked and what remains uncertain.
